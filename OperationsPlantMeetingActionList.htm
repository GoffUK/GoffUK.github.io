PM495/3

Investigate and create plan to remove the standing FGC-SRV alarms present in J2.

P Pool emailed P card to request removal of unused alarms, will chase up. 27/10/16: P Card has agreed it is an issue. Chasing continues. 03/11/16: P Card still being chased.

Pinned down P Card: It was noted that any alarms present in the zone FGC‑SRV cause a (response) bit to be set, this bit can be checked by other software, e.g. Level-1 or Countdown to warn operators. When all the alarms in the zone are cleared then a converse response is to clear the bit. In the alarm definition for this particular response an alarm message was generated, hence we had this standing alarm always present when there were no alarms. The definition has now been modified so the clear the bit response is still generated but no message is displayed. This technique is used in other alarm zones.
